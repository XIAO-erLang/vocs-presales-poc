# 源解 POC v0.5

## 当前版本定位

源解是一个环境工程技术支持与资源协同平台。

项目以 VOCs 废气治理为第一个切入口，提供计算工具、标准模板、初步方案框架、工程师对接、机械设计协作与供应商资料库，帮助客户、初级环保工程师、售前人员和项目经理更快完成项目前期判断、技术沟通与资源匹配。

当前版本目标不是完整商业系统，而是验证“工具 + 资料 + mock 支付 + 合作入口 + 资源协同”的平台骨架。

## 六大板块

1. 计算工具：面向 VOCs、废气治理与环保设备前期估算，提供可复用的售前计算工具。
2. 标准模板：沉淀工况表、现场勘查清单、报价前风险提示、客户沟通问题库等标准资料。
3. 初步方案框架：基于项目基础信息，生成用于售前沟通和内部讨论的初步方案框架，不替代正式设计。
4. 工程师对接：协助客户将结构化项目信息交给合适工程师，发起一次有效的前期技术对接。
5. 机械设计协作：面向环保设备、非标结构、风管、支架、平台、箱体等需求，预留机械设计师协作入口。
6. 优质供应商资料库：整理经初步审核的供应商资料，展示其产品能力、适用场景与需谨慎场景，不作效果担保。

## 平台商业边界

源解赚的是“效率服务费”和“工程知识资产费”，不抽客户与工程师、机械设计师或供应商之间后续深度服务的大单金额。

平台提供项目信息整理、前期判断辅助、工具模板、方案框架、工程师匹配、机械设计协作入口、供应商资料整理和首次对接协助。

平台不参与客户与工程师、机械设计师或供应商之间后续深度服务的具体报价、合同履行、项目实施和工程责任。

后续如需正式方案、图纸、计算书、现场服务、施工、调试、检测或长期顾问服务，由客户与对应服务方另行确认服务范围、费用、周期、交付物和责任边界。

正式经营主体上线后，平台可按规则提供订单记录和相应服务发票，便于企业客户内部报销和资料留档。

## 当前边界

本平台内容仅用于前期判断、售前估算、学习参考和内部沟通，不替代正式工程设计、施工图、检测验收或达标承诺。

当前版本不做：

- 真实支付扣款。
- Supabase 或其他真实数据库。
- 真实邮件发送。
- 用户登录。
- 用户后台。
- 工程师后台。
- 供应商审核后台。
- 附件上传。
- 真实工程师分账。
- 真实供应商广告投放。

## v0.4 支付与订单骨架

当前版本保留统一的 order / checkout / payment / entitlement 结构，支持：

1. 计算工具单次使用。
2. 计算工具年费会员。
3. 标准模板购买。
4. 初步方案框架购买。
5. 工程师对接订单。
6. 供应商对接占位。
7. 供应商年费入驻申请与人工审核占位。

第一阶段只使用 Mock Payment：

- mock 支付成功。
- mock 支付失败。
- mock 支付取消。
- 支付成功后 mock 开通权益。
- 模板、方案和工程师对接的邮件交付预览。
- 工程师 30 分钟优先接单、候选工程师、24 小时退款、72 小时完成确认的流程 UI mock。

真实支付接入说明见 `PAYMENT_SETUP.md`。

## 供应商审核制

源解供应商资料库不是广告位，不是谁付费谁展示。

供应商需要经过资料审核后，才可能进入资料库。审核重点包括：

- 主营产品/服务是否清晰。
- 适用行业是否清晰。
- 适用工况是否清晰。
- 不适用/需谨慎场景是否说明。
- 是否有代表案例类型。
- 服务区域是否明确。
- 资料是否完整。
- 是否存在夸大宣传或不当承诺。

供应商入驻费用当前不写死。审核通过后，根据展示范围、资料整理深度和合作方式另行确认。

## 内容共建伙伴

内容共建伙伴可以共建计算工具、标准模板、工况表、检查清单、客户沟通问题库、方案框架、案例复盘、设备选型资料和行业方法论文章。

内容共建伙伴可根据内容贡献类型、销售情况和双方确认规则，参与工具、模板、资料包或方案模块的收益分成。具体收益比例、结算周期、授权范围和署名方式，以双方确认的合作规则为准。

## 本地运行

```bash
npm install
npm run dev
```

## 构建

```bash
npm run build
```

如果本机遇到 `.next` 写入权限问题，可临时使用：

```powershell
$env:NEXT_USE_LOCAL_DIST_DIR='1'
npm.cmd run build
```

Vercel 不需要设置 `NEXT_USE_LOCAL_DIST_DIR`，保持默认构建即可。
